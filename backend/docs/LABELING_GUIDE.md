# 데이터 라벨링 전략 및 가이드

## 📋 라벨링 목표

- **총 라벨링 목표**: 최소 1,000건
- **합격 라벨**: 500건 이상
- **불합격 라벨**: 500건 이상
- **균형 비율**: 50:50 (합격:불합격)

---

## 🎯 라벨링 기준

### 합격 (is_passed = true)
다음 중 **3개 이상** 해당하면 합격으로 라벨링:

1. **구체적인 경험 서술**
   - 수치/데이터로 성과 증명
   - 구체적인 프로젝트명, 업무명 언급
   - Before/After 비교 명확

2. **직무 연관성**
   - 지원 직무와 경험이 명확히 일치
   - 요구 스킬/역량 명시적 언급
   - 직무 이해도 높음

3. **논리적 구성**
   - 서론-본론-결론 명확
   - STAR 기법 활용 (Situation, Task, Action, Result)
   - 자연스러운 흐름

4. **차별화 요소**
   - 독특한 경험/관점 제시
   - 회사/직무에 대한 깊은 이해
   - 구체적인 기여 방안 제시

5. **문장 완성도**
   - 맞춤법/문법 오류 없음
   - 적절한 어휘 선택
   - 읽기 쉬운 문장

### 불합격 (is_passed = false)
다음 중 **2개 이상** 해당하면 불합격으로 라벨링:

1. **추상적/일반적 서술**
   - "열심히", "노력했다" 등 모호한 표현
   - 구체적 사례 없음
   - 누구나 쓸 수 있는 내용

2. **직무 불일치**
   - 지원 직무와 무관한 경험만 나열
   - 요구 역량 언급 없음
   - 직무 이해 부족

3. **비논리적 구성**
   - 문단 간 연결 어색
   - 주제 혼재
   - 흐름 끊김

4. **차별화 부족**
   - 템플릿 느낌
   - 회사 리서치 부족
   - 성의 없는 느낌

5. **문장 문제**
   - 맞춤법/문법 오류 다수
   - 부적절한 어휘
   - 읽기 어려움

### 건너뛰기 (Skip)
다음 경우 건너뛰기:

- 판단 기준 애매 (합격도 불합격도 아닌 중간)
- 내용이 너무 짧거나 불충분
- 특수한 케이스 (예: 외국어 혼재)

## 🔄 라벨링 프로세스

### 1단계: 빠른 스캔 (10초)
- 제목, 회사, 직무 확인
- 전체 길이 확인
- 첫 문단 읽기

### 2단계: 전체 읽기 (30~60초)
- 전체 내용 정독
- 핵심 경험/성과 파악
- 논리적 흐름 확인

### 3단계: 판단 (10초)
- 위 기준에 따라 합격/불합격 결정
- 애매하면 건너뛰기
- 즉시 라벨링

### 시간 목표
- **1건당 평균 1분**
- **시간당 약 50~60건**
- **총 20시간 내 1,000건 완료**

## 📊 라벨링 관리

### 일일 목표
- **최소 50건/일**
- **권장 100건/일**
- **최대 200건/일** (과도한 라벨링 방지)

### 품질 관리
- **매 10건마다 휴식**
- **매 50건마다 일관성 체크**
- **매 100건마다 분포 확인** (합격:불합격 비율)

### 진행률 모니터링
```
목표: 1,000건
현재: X건 (X%)
합격: Y건
불합격: Z건
비율: Y:Z (목표 50:50)
```

## 🎓 라벨링 예시

### 예시 1: 합격
```
[제목] 삼성전자 마케팅 직무 지원
[회사] 삼성전자
[직무] 마케팅

[내용]
저는 3년간 스타트업에서 디지털 마케팅을 담당하며, 
월 평균 ROI를 150%에서 320%로 향상시킨 경험이 있습니다.

특히 SNS 광고 캠페인을 기획하여...
(구체적인 성과, STAR 기법, 직무 연관성 높음)
```
→ **합격**: 구체적 성과, 수치 제시, 직무 연관성

### 예시 2: 불합격
```
[제목] 입사 지원
[회사] 현대자동차
[직무] 생산

[내용]
저는 항상 열심히 노력하는 사람입니다.
회사에 입사하면 최선을 다하겠습니다.
...
```
→ **불합격**: 추상적, 경험 없음, 성의 부족

## 🛠 도구 사용법

### 웹 인터페이스
1. http://localhost:3000/labeling 접속
2. 자기소개서 읽기
3. 합격/불합격/건너뛰기 버튼 클릭
4. 자동으로 다음 항목 로드

### 단축키 (추후 구현 예정)
- `1` 또는 `←`: 불합격
- `2` 또는 `↓`: 건너뛰기
- `3` 또는 `→`: 합격
- `R`: 새로고침

### API 사용 (선택)
```bash
# 다음 항목 조회
curl http://localhost:9000/api/labeling/next

# 라벨링
curl -X POST http://localhost:9000/api/labeling/label \
  -H "Content-Type: application/json" \
  -d '{"cover_letter_id": 1, "is_passed": true}'

# 통계 확인
curl http://localhost:9000/api/labeling/stats
```

## ⚠️ 주의사항

1. **편향 방지**
   - 특정 회사에 편향되지 않도록 주의
   - 직무별로 고르게 라벨링
   - 합격 라벨에 너무 인색하지 않기

2. **일관성 유지**
   - 라벨링 기준을 일관되게 적용
   - 피로할 때는 휴식 (품질 저하 방지)
   - 매일 같은 시간대에 라벨링 권장

3. **품질 우선**
   - 속도보다 정확도 우선
   - 애매하면 건너뛰기
   - 나중에 다시 검토 가능
